# Security

Projekt ma charakter edukacyjny (MVP). Jeśli chcesz używać go „produkcyjnie”, rozważ:
- trzymanie klucza szyfrującego po stronie klienta (client-side crypto),
- trwałe sesje połączone z bezpiecznym magazynem kluczy (np. OS keyring),
- 2FA i politykę blokad,
- pełny model uprawnień oraz logowanie zdarzeń.

Zgłoszenia problemów bezpieczeństwa: utwórz prywatne issue lub skontaktuj się z zespołem.
